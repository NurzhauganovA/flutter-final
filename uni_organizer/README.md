# UniOrganizer

Кроссплатформенное мобильное приложение на Flutter для организации учебного процесса студентов. Объединяет в себе функции расписания, таск-менеджера и заметок с облачной синхронизацией.

## Основной функционал

* **Dashboard**: Сводная статистика успеваемости, прогресс выполнения задач и обзор пар на текущий день.
* **Управление задачами**: Добавление академических задач с дедлайнами. Фильтрация по статусу, датам и категориям. Цветовая маркировка приоритетов.
* **Расписание занятий**: Просмотр пар по дням недели с сортировкой по времени. Поддержка разных типов занятий (лекции, семинары, лабы).
* **Заметки**: Создание текстовых записей с возможностью цветового выделения и закрепления (Pin) важных заметок вверху списка.
* **Профиль**: Редактирование данных студента (имя, группа, специальность).
* **Уведомления**: Локальные напоминания о дедлайнах (за сутки и в день сдачи) и парах (за 5 минут до начала).

## Технологический стек

* **Фреймворк**: Flutter (Dart)
* **Backend**: Firebase (BaaS)
* **База данных**: Cloud Firestore
* **Аутентификация**: Firebase Auth
* **Локальные уведомления**: flutter_local_notifications

## Архитектура

Проект построен на архитектурном паттерне **MVVM (Model-View-ViewModel)**.

* **Model**: Описывает структуру данных (`Task`, `ScheduleItem`, `Note`, `UserProfile`).
* **View**: Отвечает за отображение интерфейса (экраны и виджеты). Пассивно реагирует на изменения состояния.
* **ViewModel (Providers)**: Содержит бизнес-логику, связывает UI с сервисами и управляет состоянием через пакет `Provider`.

## Работа с данными

Используется **Cloud Firestore** с иерархической структурой для обеспечения приватности.

**Схема данных:**
* Коллекция `users` (документ по `uid` пользователя)
    * Подколлекция `tasks`: Хранение задач, статусов и дедлайнов.
    * Подколлекция `schedule`: Хранение повторяющегося расписания.
    * Подколлекция `notes`: Хранение личных записей.

Взаимодействие с базой происходит через **Streams**, что обеспечивает мгновенное обновление интерфейса при изменении данных.

## Сервисы

* **AuthService**: Обработка регистрации, входа (Email/Password) и выхода. При регистрации автоматически создает профиль в БД.
* **DatabaseService**: Обертка над Firestore SDK. Реализует CRUD-операции для всех сущностей.
* **NotificationService**: Инициализация и планирование уведомлений с учетом часового пояса устройства.
